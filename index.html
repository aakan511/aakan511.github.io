Building the Tractyl Manuform - One of the Strangest Keyboards on the Internet

Above is a fully assembled image of the Tractyl Manuform 5x6. Building this keyboard took around 100 hours of work. First, the curved design of the keyboard deck made it impossible to use a PCB, so I had to hand-solder the diodes to the hot-swappable key-switch holders and then use wires to create the keyboard matrix. Next, I had to somehow make the two halves communicate. For this, I used serial communication over a generic audio cable because the data throughput is small. Programming the Raspberry Pi Pico controllers was a challenge. I decided to use Python instead of C because it would allow me to change the code on the microcontrollers rather than recompiling firmware every time I made a change. The hardest part of the code was figuring out how to get the trackball to communicate with my computer, which I eventually figured out.

Motivations:

  • Create an ergonomic keyboard to preserve wrist and shoulder health
  • Integrate trackball in order to improve efficiency by not having to reach for a mouse
  • Improve typing speed by having thumb clusters
  • Build a "final" keyboard that would not be replaced for a long time

Kibo Robot Programming Challenge

Technologies & STEM Concepts Used:

  • Java
  • OpenCV image recognition
  • Android Studio
  • Quaternions
  • Path planning

In April 2023, I led Team Salcedo for our second year competing in the Kibo Robot Programming Challenge. This year we won the national round, which allowed us to send our code to the ISS and compete internationally as NASA's representative. Unfortunately, an unforseen bug caused our program to crash right before finishing. Learning from this mistake will propel us to greater sucess next year! As the captain, I was responsible for organizing meetings and setting deadlines for each part of the code and helping my teammates as best I could. We were also fortunate enough to be able to work with some college students from Sri Lanka, which was an amazing experience for me because I learned a lot about their culture and programming, and I made some friends on the other side of the globe! In the competition, the main goal is to use image recognition to find and patch air leaks on the ISS.

The image above is an examble image from the simulator for the Astrobee robot that we programed during this challenge. Processing this image took several steps. First, we had to remove the distortion caused by the Astrobee's wide-angle lens. Next, we scanned and found the location of the Aruco markers in order to calculate our distance from the target. The small white dot in the center of the target represents our robot's laser. Clearly, this run was successful beceause we hit the target in the bullseye!

Future Goals:

While this is an impressive result, I believe that we can do better, which is one of the major reasons that I am working on founding a Kibo Robot Programming Challenge team at Georgia Tech. I am really excited to teach my fellow yellow jackets about programming free-floating robots in microgravity and share this amazing opportunity with them!

The most interesting feature of this website (for me) is the background. If you scroll up and down, you can see that a set of stars move in the background. If you refresh the website, you can notice that the stars actually move location. The stars in the background are 3D rendered using JavaScript and are placed at random positions every time the website is loaded.

This project has been really intersting to me because it lies in the intersection of art and science since it involves both design and coding. One thing I learned in this project is that more features is not always better. I initially thought this was a novel addition that showed my familiarity with HTML and CSS, but after a large number of users told me that this color change was distracting and unappealing, I decided to remove this feature.

My 3D Printing Journey

Above is a picture of the latest iteration of my CR10S4's hotend and extruder. This new design moves the extruder(motor on top) very close to the hotend, which not only allows faster printing but also gives me the ability to print flexible materials such as TPU. The mounting mechanisms are based on another design on Thingiverse.

Here is a picture of my Evnovo Sidewinder X1. The white plastic piece on the hotend is a new fan mount that cools the molten plastic that comes out of the hotend faster, which allows quicker prints. Also, I replaced the mainboard of this system and reprogrammed it to use Klipper, which connects the printer to the internet and increases print speed and quality, while reducing noise. The pieces of styrofoam under the printer serve to dampen vibrations, which reduces the printer's noise.
